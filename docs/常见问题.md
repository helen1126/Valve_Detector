# 常见问题

本文档汇总了阀门角度检测系统使用过程中的常见问题及解决方法。

## 环境搭建问题

### CUDA 版本不匹配

**现象**：安装 PyTorch 后 `torch.cuda.is_available()` 返回 `False`，或运行时报 CUDA 版本不兼容错误。

**原因**：PyTorch 编译时的 CUDA 版本与系统安装的 NVIDIA 驱动/CUDA Toolkit 版本不匹配。

**解决方法**：

1. 检查 NVIDIA 驱动版本：
   ```bash
   nvidia-smi
   ```
   确认驱动版本 ≥ 525.60.13（CUDA 12.1 最低要求）

2. 安装匹配的 PyTorch 版本：
   ```bash
   # CUDA 12.1
   conda install pytorch torchvision pytorch-cuda=12.1 -c pytorch -c nvidia

   # CUDA 11.8（如驱动不支持 12.1）
   conda install pytorch torchvision pytorch-cuda=11.8 -c pytorch -c nvidia
   ```

3. 验证 CUDA 可用：
   ```python
   import torch
   print(torch.cuda.is_available())  # 应输出 True
   print(torch.version.cuda)         # 应输出对应 CUDA 版本
   ```

### 依赖冲突

**现象**：`pip install -r requirements.txt` 时报依赖版本冲突。

**解决方法**：

1. 使用全新的 Conda 环境安装：
   ```bash
   conda create -n valve_detector python=3.10 -y
   conda activate valve_detector
   ```

2. 先安装 PyTorch，再安装其他依赖：
   ```bash
   conda install pytorch torchvision pytorch-cuda=12.1 -c pytorch -c nvidia
   pip install -r requirements.txt
   ```

3. 如仍有冲突，尝试逐个安装关键依赖：
   ```bash
   pip install timm albumentations pytorch-lightning fastapi streamlit
   ```

### Python 版本不兼容

**现象**：部分库安装失败或运行时报语法错误。

**解决方法**：确保使用 Python 3.10，不支持 3.11+ 或 3.9- 版本。

```bash
python --version  # 确认版本为 3.10.x
```

### OpenCV 依赖缺失（云服务器）

**现象**：在云服务器上运行时报错：
```
ImportError: libXext.so.6: cannot open shared object file: No such file or directory
```

**原因**：`opencv-python` 依赖 X11 图形界面库，云服务器通常是无头环境。

**解决方法**：使用 `opencv-python-headless` 替代 `opencv-python`

```bash
# 卸载当前的 opencv-python
pip uninstall opencv-python -y

# 安装 headless 版本
pip install opencv-python-headless
```

或者在 `requirements.txt` 中替换：
```txt
# opencv-python>=4.8.0
opencv-python-headless>=4.8.0
```

### HuggingFace 网络不可达（云服务器）

**现象**：训练时报错：
```
'[Errno 101] Network is unreachable' thrown while requesting HEAD `https://huggingface.co/timm/convnext_base.fb_in22k_ft_in1k/resolve/main/model.safetensors`
```

**原因**：云服务器无法访问 huggingface.co 下载预训练模型。

**解决方法**：使用国内镜像

```bash
# 设置环境变量
export HF_ENDPOINT=https://hf-mirror.com

# 然后运行训练
python train.py --model convnext_base --data_dir ./dataset --view all_view
```

永久配置（添加到 `~/.bashrc`）：
```bash
echo 'export HF_ENDPOINT=https://hf-mirror.com' >> ~/.bashrc
source ~/.bashrc
```

### TensorBoard 启动失败

**现象**：运行 `tensorboard --logdir ./logs/tensorboard` 报错：
```
ModuleNotFoundError: No module named 'pkg_resources'
```

**原因**：`setuptools 82.0.1` 版本移除了 `pkg_resources` 模块。

**解决方法**：降级 setuptools 到旧版本

```bash
pip install "setuptools<71"
```

## 训练过程问题

### 显存不足（OOM）

**现象**：训练时报 `CUDA out of memory` 错误。

**解决方法**：

1. **减小批大小**：最直接有效的方法
   ```bash
   python train.py --batch_size 16   # 从 32 减小到 16
   python train.py --batch_size 8    # 进一步减小
   ```

2. **减小图像尺寸**：
   ```bash
   python train.py --image_size 224  # 从 384 减小到 224
   ```

3. **使用更小的模型**：
   ```bash
   python train.py --model resnet50              # 替代 convnext_base
   python train.py --model efficientnet_b4       # 替代 swin_base
   ```

4. **减少数据加载线程**：
   ```bash
   python train.py --num_workers 2   # 从 4 减小到 2
   ```

5. **关闭混合精度**：在 `config/train_config.yaml` 中将 `precision` 改为 `32`（不推荐，会增大显存占用）

### 训练收敛慢

**现象**：训练多个 epoch 后损失下降非常缓慢。

**解决方法**：

1. **提高学习率**：
   ```bash
   python train.py --lr 2e-4   # 从 1e-4 提高到 2e-4
   ```

2. **检查数据质量**：确认图片命名格式正确（`编号_角度.后缀`），角度标签无误

3. **增大余弦退火周期**：在 `config/train_config.yaml` 中增大 `T_max`
   ```yaml
   scheduler:
     T_max: 100   # 从 50 增大到 100
   ```

4. **使用预训练权重**：确认 `model_config.yaml` 中 `pretrained: true`

5. **增加数据量**：收集更多不同角度的阀门图片

### 过拟合

**现象**：训练集 MAE 很低但验证集 MAE 较高，两者差距大。

**解决方法**：

1. **增加数据增强强度**：在 `config/data_config.yaml` 中提高各增强方法的概率 `p` 值
   ```yaml
   augmentation:
     random_brightness_contrast:
       p: 0.7   # 从 0.5 提高
     gaussian_blur:
       p: 0.5   # 从 0.3 提高
     coarse_dropout:
       p: 0.5   # 从 0.3 提高
   ```

2. **增大 Dropout**：在 `config/model_config.yaml` 中增大 `dropout`
   ```yaml
   dropout: 0.3   # 从 0.2 增大到 0.3
   ```

3. **增大权重衰减**：
   ```bash
   python train.py --weight_decay 5e-4   # 从 1e-4 增大到 5e-4
   ```

4. **使用更小的模型**：大模型更容易过拟合，可尝试 `resnet50` 或 `efficientnet_b4`

5. **增加数据量**：最根本的解决方法

6. **早停机制**：确认 `early_stopping.enabled: true`，避免过度训练

## 预测精度问题

### MAE 偏高

**现象**：评估时 MAE > 1°，未达到精度目标。

**解决方法**：

1. **使用更大模型**：
   ```bash
   python train.py --model convnext_large   # 从 convnext_base 升级
   python train.py --model swin_large       # 或使用 Swin-Large
   ```

2. **增大图像尺寸**：
   ```bash
   python train.py --image_size 512   # 从 384 增大到 512
   ```

3. **延长训练时间**：
   ```bash
   python train.py --epochs 300   # 从 200 增大到 300
   ```

4. **调整损失权重**：增大 MSE 权重以减少大误差
   ```bash
   python train.py --mae_weight 0.5 --mse_weight 0.5
   ```

5. **启用图像优化**：对光照不佳的图片启用预处理优化
   ```bash
   python predict.py --model_path ./weights/last.ckpt --input ./test.jpg --optimize
   ```

6. **检查数据质量**：排查标注错误、模糊图片、异常角度值

### 角度偏移（系统性偏差）

**现象**：预测角度整体偏大或偏小。

**原因**：训练数据分布不均衡，某些角度区间样本过少。

**解决方法**：

1. **补充数据**：在样本少的角度区间增加更多图片
2. **查看评估报告**：运行 `evaluate.py` 生成报告，检查各角度区间的 MAE 分布
   ```bash
   python evaluate.py --model_path ./weights/last.ckpt --output_dir ./logs/evaluation
   ```
3. **调整角度范围**：如果实际应用角度范围较窄，可在 `data_config.yaml` 中调整 `angle_min` 和 `angle_max`

### 特定角度预测不准

**现象**：某些角度区间（如 0°~5° 或 70°~80°）预测误差较大。

**解决方法**：

1. **补充极端角度数据**：在 0°~5° 和 70°~80° 区间增加更多样本
2. **使用区间加权损失**：对样本少的区间增大损失权重
3. **查看误差分布图**：评估脚本会生成 `interval_errors.png`，帮助定位问题区间

### Side 视角 MAE 过高

**现象**：top 视角 MAE <1°，但 side 视角 MAE >6°，差距巨大。

**原因**：side 视角存在透视变形、遮挡、镜头畸变，学习难度远高于 top 视角，且样本量通常较少。

**解决方法**：

1. **启用视角加权 + 难样本挖掘**（推荐）：
   ```bash
   # side_weight=5.0 提升 side 样本权重，focal_gamma=2.0 聚焦难样本
   python train.py --model convnext_base --view all_view --side_weight 5.0 --focal_gamma 2.0
   ```

2. **两阶段训练**：
   ```bash
   # 先 all_view 训练基础模型，再用 side_view 微调
   python train.py --model convnext_base --view all_view --stage2 --stage2_epochs 30 --stage2_lr 1e-5
   ```

3. **组合策略**（效果最佳）：
   ```bash
   python train.py --model convnext_base --view all_view \
       --side_weight 5.0 --focal_gamma 2.0 \
       --stage2 --stage2_epochs 30 --stage2_lr 1e-5
   ```

4. **提升 side_weight**：side 视角极难时可进一步提升权重
   ```bash
   python train.py --model convnext_base --view all_view --side_weight 10.0
   ```

5. **监控分视角指标**：在 TensorBoard 中观察 `val_side_mae` 和 `val_top_mae` 的变化趋势

### 远距离拍摄预测不准

**现象**：拍摄距离过远时，阀门在画面中占比过小，预测角度极不精确。

**原因**：远距离拍摄时阀门区域过小，模型主要看到背景信息，缺乏阀门细节特征。

**解决方法**：

1. **启用智能裁剪**（推荐）：
   ```bash
   # 自动检测阀门区域并裁剪放大
   python predict.py --model_path ./weights/last.ckpt --input ./test.jpg --smart_crop
   ```

2. **启用多尺度推理**（精度最高）：
   ```bash
   # 原图+裁剪图加权融合
   python predict.py --model_path ./weights/last.ckpt --input ./test.jpg --multi_scale
   ```

3. **API 调用时启用**：
   ```bash
   curl -X POST "http://localhost:8000/predict?smart_crop=true" -F "file=@test.jpg"
   curl -X POST "http://localhost:8000/predict?multi_scale=true" -F "file=@test.jpg"
   ```

4. **前端使用**：在侧边栏勾选"智能裁剪"或"多尺度推理"

5. **训练时也启用智能裁剪**：在 `config/data_config.yaml` 中启用 `smart_crop` 增强，让模型在训练时见到裁剪放大后的图像，减少训练-推理分布不一致

### 模型架构不匹配

**现象**：加载 checkpoint 时报错 `size mismatch for ...`，提示维度不匹配。

**原因**：训练时使用的模型架构与预测时指定的架构不一致（如训练用 convnext_large，预测时默认用 convnext_base）。

**解决方法**：

系统已自动从 checkpoint 中检测模型架构，无需手动指定。如果仍有问题：

```bash
# 方式1：通过命令行参数指定正确的模型架构
python predict.py --model_path ./weights/best.ckpt --model_name convnext_large --input ./test.jpg

# 方式2：通过环境变量指定（API 服务）
export MODEL_NAME=convnext_large
```

## 部署问题

### 端口占用

**现象**：启动 API 或前端时提示 `Address already in use`。

**解决方法**：

```bash
# Windows：查找并终止占用端口的进程
netstat -ano | findstr :8000
taskkill /PID <进程ID> /F

# 或更换端口启动
uvicorn api.main:app --host 0.0.0.0 --port 8001
streamlit run frontend/app.py --server.port 8502
```

### 模型加载失败

**现象**：API 启动时日志提示模型加载失败，`/health` 返回 `model_not_loaded`。

**排查步骤**：

1. **检查文件路径**：确认权重文件存在于指定路径
   ```bash
   dir .\weights\last.ckpt
   ```

2. **检查模型架构匹配**：`MODEL_NAME` 环境变量必须与训练时使用的架构一致
   ```bash
   export MODEL_NAME=convnext_base   # 必须与训练时 --model 参数一致
   ```

3. **检查权重格式**：支持 `.ckpt`（PyTorch Lightning 格式）和 `.pth`（纯 PyTorch 格式）

4. **查看详细日志**：API 启动时会输出详细的错误信息

5. **手动验证**：先用 `predict.py` 验证模型能否正常加载
   ```bash
   python predict.py --model_path ./weights/last.ckpt --input ./test.jpg
   ```

### API 响应超时

**现象**：请求 API 时响应时间过长。

**排查方向**：

1. **检查设备**：确认模型运行在 GPU 上
   ```bash
   curl http://localhost:8000/info
   # 查看 device 字段，应为 "cuda:0" 而非 "cpu"
   ```

2. **检查图片大小**：过大的图片会增加传输和预处理时间

3. **使用 ONNX 加速**：
   ```bash
   # 导出 ONNX 模型后使用
   python predict.py --model_path ./weights/model.onnx --input ./test.jpg --onnx
   ```

4. **关闭图像优化**：图像优化流水线会增加处理时间

### Streamlit 界面无法加载模型

**现象**：前端 Demo 中点击"加载模型"后报错。

**解决方法**：

1. 确认模型路径输入正确（支持绝对路径和相对路径）
2. 确认模型架构选择与权重文件匹配
3. 查看 Streamlit 终端输出的错误信息
4. 尝试使用命令行预测验证模型可用性

### ONNX 导出失败

**现象**：导出 ONNX 模型时报错。

**解决方法**：

1. 确认模型已正确加载并处于 eval 模式
2. 检查 `opset_version` 是否支持当前模型结构（推荐 17）
3. 确认 `input_size` 与训练时的 `image_size` 一致
4. 部分 Swin Transformer 变体可能需要更高的 opset_version

### HTTPS 相关问题

**现象**：HTTPS 前端调用 HTTP API 时，浏览器控制台报 "Mixed Content" 错误，请求被拦截。

**原因**：HTTPS 页面只能调用 HTTPS 接口，浏览器会拦截所有 HTTP 请求（混合内容策略）。

**解决方法**：启用 API 的 HTTPS 模式，详见[部署指南 - HTTPS 部署](部署指南.md#https-部署)。

---

**现象**：访问 HTTPS API 时浏览器提示 "您的连接不是私密连接"。

**原因**：使用了自签名证书，浏览器不信任。

**解决方法**：
- 开发环境：点击"高级"→"继续前往"即可
- 生产环境：使用 Let's Encrypt 等受信任 CA 签发的证书

---

**现象**：启动 HTTPS 模式后，HTTP 端口（8000）和 HTTPS 端口（8443）都能访问 API。

**原因**：HTTPS 模式下会同时启动 HTTP 和 HTTPS 两个 API 服务，提供相同接口，方便不同场景接入（如 Cloudflare 隧道指向 HTTP 端口，浏览器直连 HTTPS 端口）。

**解决方法**：这是正常行为。如只需单端口，可使用 `uvicorn` 命令行方式启动。

---

**现象**：通过 Cloudflare 隧道访问时浏览器报 `ERR_TOO_MANY_REDIRECTS`（重定向次数过多）。

**原因**：Cloudflare 隧道指向了后端的重定向端口，导致浏览器↔Cloudflare↔后端之间形成重定向死循环。

**解决方法**：
1. 确认使用的是双服务模式（HTTP 端口不重定向，直接提供 API）
2. Cloudflare 隧道服务类型保持 **HTTP**，URL 指向 `http://localhost:8000`
3. Cloudflare 边缘会自动提供 HTTPS 给浏览器，无需在后端配置 SSL 证书
4. 如仍需后端 HTTPS，确保 `api/main.py` 启动的是双服务模式（HTTP 端口运行 `api.main:app`，非重定向服务）

### 视频预测问题

**现象**：视频抽帧预测时报错或结果异常。

**解决方法**：

1. **视频格式不支持**：确保视频为 mp4/avi/mov/mkv 格式，必要时用 ffmpeg 转码
   ```bash
   ffmpeg -i input.wmv -c:v libx264 output.mp4
   ```

2. **视频文件损坏**：用 ffprobe 检查视频完整性
   ```bash
   ffprobe -v error -show_format input.mp4
   ```

3. **抽帧频率过高导致处理慢**：降低 fps 或增大 frame_interval
   ```bash
   # 降低到每秒 1 帧
   python predict.py --model_path ./weights/last.ckpt --input ./test.mp4 --fps 1
   ```

4. **输出视频无法播放**：标注视频使用 mp4v 编码，确保播放器支持，或用 ffmpeg 转码
   ```bash
   ffmpeg -i predicted_video.mp4 -c:v libx264 predicted_video_h264.mp4
   ```

5. **显存不足**：视频预测时会逐帧处理，确保 GPU 显存充足，或使用 ONNX 模型减少显存占用
