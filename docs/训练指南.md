# 训练指南

本文档详细介绍阀门角度检测模型的训练流程，包括数据集准备、配置文件说明、训练命令、监控方法和调优建议。

## 数据集准备

### 目录结构

数据集应放置在 `./dataset` 目录下，按视角分子目录组织：

```
dataset/
├── all_view/           # 多视角图像（推荐，数据量最大）
│   ├── 0001_6.7.jpg
│   ├── 0002_6.7.jpg
│   ├── 0003_11.5.jpg
│   └── ...
├── side_view/          # 侧视图像
│   ├── 0001_8.2.jpg
│   └── ...
└── top_view/           # 俯视图像
    ├── 0001_3.4.jpg
    └── ...
```

### 命名格式

图片文件命名格式为：**编号_角度.后缀**

```
{编号}_{角度}.{jpg|jpeg|png|bmp}
```

- **编号**：4位数字编号，如 `0001`、`0233`
- **角度**：浮点数，表示阀门开度角度，单位为度（°），范围 0.0~80.0
- **后缀**：支持 `.jpg`、`.jpeg`、`.png`、`.bmp` 格式

示例：
- `0001_6.7.jpg` — 编号0001，角度6.7°
- `0233_25.jpg` — 编号0233，角度25.0°
- `0659_1.0.png` — 编号0659，角度1.0°

### 数据划分

系统自动按配置比例划分训练集、验证集和测试集（默认 8:1:1），通过随机种子确保可复现。划分比例在 `config/data_config.yaml` 中配置：

```yaml
train_ratio: 0.8
val_ratio: 0.1
test_ratio: 0.1
seed: 42
```

## 配置文件参数详解

### data_config.yaml — 数据配置

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `data_dir` | `./dataset` | 数据集根目录 |
| `view` | `all_view` | 视角选择：`all_view` / `top_view` / `side_view` |
| `image_size` | `384` | 输入图像尺寸 |
| `train_ratio` | `0.8` | 训练集比例 |
| `val_ratio` | `0.1` | 验证集比例 |
| `test_ratio` | `0.1` | 测试集比例 |
| `seed` | `42` | 随机种子 |
| `angle_min` | `0.0` | 最小角度 |
| `angle_max` | `80.0` | 最大角度 |
| `num_workers` | `4` | 数据加载线程数 |
| `pin_memory` | `true` | 是否锁页内存 |

**预处理配置** (`preprocess` 子项)：

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `enabled` | `true` | 是否启用预处理 |
| `brightness` | `1.0` | 亮度调整系数 |
| `contrast` | `1.0` | 对比度调整系数 |
| `saturation` | `1.0` | 饱和度调整系数 |
| `denoise` | `false` | 是否启用去噪 |
| `denoise_strength` | `10` | 去噪强度 |
| `clahe` | `false` | 是否启用 CLAHE 光照校正 |
| `clahe_clip_limit` | `2.0` | CLAHE 裁剪限制 |
| `clahe_grid_size` | `8` | CLAHE 网格大小 |

**数据增强配置** (`augmentation` 子项)：

| 增强方法 | 参数 | 默认值 | 说明 |
|----------|------|--------|------|
| `random_brightness_contrast` | `brightness_limit` / `contrast_limit` / `p` | `0.2` / `0.2` / `0.5` | 随机亮度对比度 |
| `hue_saturation_value` | `hue_shift_limit` / `sat_shift_limit` / `val_shift_limit` / `p` | `20` / `30` / `20` / `0.5` | 色调饱和度变化 |
| `gaussian_blur` | `blur_limit` / `p` | `7` / `0.3` | 高斯模糊 |
| `gauss_noise` | `var_limit` / `p` | `[10.0, 50.0]` / `0.3` | 高斯噪声 |
| `random_resized_crop` | `scale` / `ratio` / `p` | `[0.8, 1.0]` / `[0.9, 1.1]` / `0.5` | 随机裁剪缩放 |
| `coarse_dropout` | `max_holes` / `max_height` / `max_width` / `p` | `8` / `32` / `32` / `0.3` | 随机遮挡 |
| `sharpen` | `alpha` / `lightness` / `p` | `[0.2, 0.5]` / `[0.5, 1.0]` / `0.3` | 锐化 |
| `rgb_shift` | `r/g/b_shift_limit` / `p` | `15` / `0.3` | RGB 通道偏移 |
| `channel_shuffle` | `p` | `0.1` | 通道混洗（默认关闭） |

> **注意**：不使用旋转和翻转增强，因为会改变阀门角度的视觉表现。

### model_config.yaml — 模型配置

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `model_name` | `convnext_base` | 默认模型架构 |
| `pretrained` | `true` | 是否使用预训练权重 |
| `in_channels` | `3` | 输入通道数 |
| `angle_min` | `0.0` | 最小角度（归一化用） |
| `angle_max` | `80.0` | 最大角度（归一化用） |
| `freeze_backbone` | `false` | 是否冻结骨干网络 |
| `freeze_layers` | `0` | 冻结层数 |
| `dropout` | `0.2` | Dropout 概率 |
| `save_dir` | `./weights` | 模型保存路径 |

**可选模型架构**：

| 模型名称 | 骨干来源 | 说明 |
|----------|----------|------|
| `resnet50` | torchvision | ResNet-50 |
| `resnet101` | torchvision | ResNet-101 |
| `efficientnet_b4` | timm | EfficientNet-B4 |
| `efficientnet_b5` | timm | EfficientNet-B5 |
| `convnext_base` | timm | ConvNeXt-Base（推荐） |
| `convnext_large` | timm | ConvNeXt-Large |
| `swin_base` | timm | Swin-Base-Patch4-Window12-384 |
| `swin_large` | timm | Swin-Large-Patch4-Window12-384 |

**ONNX 导出配置** (`onnx` 子项)：

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `enabled` | `true` | 是否启用 ONNX 导出 |
| `opset_version` | `17` | ONNX 算子集版本 |
| `dynamic_axes.input.0` | `batch_size` | 输入动态批次维度 |
| `dynamic_axes.output.0` | `batch_size` | 输出动态批次维度 |

### train_config.yaml — 训练配置

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `epochs` | `200` | 最大训练轮数 |
| `batch_size` | `32` | 批大小 |

**优化器配置** (`optimizer` 子项)：

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `name` | `adamw` | 优化器名称 |
| `lr` | `1.0e-4` | 学习率 |
| `weight_decay` | `1.0e-4` | 权重衰减 |
| `betas` | `[0.9, 0.999]` | Adam beta 参数 |

**学习率调度器** (`scheduler` 子项)：

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `name` | `cosine_annealing` | 余弦退火调度 |
| `T_max` | `50` | 余弦周期 |
| `eta_min` | `1.0e-6` | 最小学习率 |

**损失函数配置** (`loss` 子项)：

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `mae_weight` | `0.7` | L1 (MAE) 损失权重 |
| `mse_weight` | `0.3` | L2 (MSE) 损失权重 |

组合损失公式：`Loss = 0.7 × L1 + 0.3 × L2`

**早停配置** (`early_stopping` 子项)：

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `enabled` | `true` | 是否启用早停 |
| `monitor` | `val_mae` | 监控指标 |
| `patience` | `10` | 容忍轮数 |
| `mode` | `min` | 最小化监控指标 |
| `min_delta` | `0.01` | 最小变化量 |

**检查点配置** (`checkpoint` 子项)：

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `monitor` | `val_mae` | 监控指标 |
| `mode` | `min` | 最小化监控指标 |
| `save_top_k` | `3` | 保存最优的 K 个模型 |
| `filename` | `valve-{epoch:03d}-{val_mae:.4f}` | 文件名格式 |
| `save_last` | `true` | 是否保存最后一个 epoch 的模型 |

**其他配置**：

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `gradient_clip.enabled` | `true` | 是否启用梯度裁剪 |
| `gradient_clip.max_norm` | `1.0` | 梯度裁剪最大范数 |
| `amp` | `true` | 混合精度训练 |
| `precision` | `16-mixed` | 精度设置 |

| `log_every_n_steps` | `10` | 日志记录频率 |

## 训练命令和参数说明

### 基本训练命令

```bash
# 使用 ConvNeXt-Base 训练（推荐，精度最优）
python train.py --model convnext_base --data_dir ./dataset --view all_view

# 使用 ResNet50 训练（速度快，适合快速验证）
python train.py --model resnet50 --epochs 100 --batch_size 16

# 使用 Swin-Base 训练
python train.py --model swin_base --view top_view --image_size 384

# 使用 EfficientNet-B4 训练
python train.py --model efficientnet_b4 --lr 5e-5 --batch_size 24
```

### 命令行参数

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `--data_dir` | `./dataset` | 数据集根目录 |
| `--view` | `all_view` | 视角选择：`all_view` / `top_view` / `side_view` |
| `--model` | `convnext_base` | 模型架构（见可选模型列表） |
| `--epochs` | `200` | 最大训练轮数 |
| `--batch_size` | `32` | 批大小 |
| `--lr` | `1e-4` | 学习率 |
| `--weight_decay` | `1e-4` | 权重衰减 |
| `--image_size` | `384` | 图像尺寸 |
| `--num_workers` | `4` | 数据加载线程数 |
| `--mae_weight` | `0.7` | MAE 损失权重 |
| `--mse_weight` | `0.3` | MSE 损失权重 |
| `--seed` | `42` | 随机种子 |
| `--resume` | `None` | 从检查点恢复训练 |
| `--config_dir` | `./config` | 配置文件目录 |

### 从检查点恢复训练

```bash
python train.py --model convnext_base --resume ./weights/valve-epoch=050-val_mae=0.8543.ckpt
```

恢复训练时，优化器状态、学习率调度器状态和训练轮数都会从检查点恢复。

## 训练过程监控

### TensorBoard

训练过程使用 TensorBoard 记录日志，启动方式：

```bash
tensorboard --logdir ./logs/tensorboard
```

然后在浏览器中访问 http://localhost:6006 查看：

- **Scalars**：`train_loss`、`train_mae`、`val_loss`、`val_mae`、`val_mse`、学习率变化曲线
- **HPARAMS**：超参数配置

### 关键监控指标

| 指标 | 说明 | 目标 |
|------|------|------|
| `train_mae` | 训练集 MAE（角度空间） | 持续下降 |
| `val_mae` | 验证集 MAE（角度空间） | ≤ 1.0° |
| `train_loss` | 训练集组合损失 | 持续下降 |
| `val_loss` | 验证集组合损失 | 同步下降 |
| `lr` | 当前学习率 | 余弦退火曲线 |

### 训练输出

模型权重保存在 `./weights/` 目录下：

- `valve-epoch=XXX-val_mae=X.XXXX.ckpt` — 最优的 Top-K 模型
- `last.ckpt` — 最后一个 epoch 的模型

## 模型调优建议

### 1. 调整学习率

- **默认学习率** `1e-4` 适用于大多数场景
- 如果训练初期损失下降过慢，可尝试提高至 `2e-4` 或 `5e-4`
- 如果训练不稳定（损失震荡），可降低至 `5e-5` 或 `1e-5`
- 调整余弦退火周期 `T_max`：增大周期可获得更平缓的学习率衰减

```bash
python train.py --model convnext_base --lr 2e-4
```

### 2. 增加数据增强

- 在 `config/data_config.yaml` 的 `augmentation` 中启用更多增强方法
- 适当提高各增强方法的概率 `p` 值
- 启用 `channel_shuffle` 增加颜色多样性
- 增大 `random_resized_crop` 的 `scale` 范围（如 `[0.7, 1.0]`）

```yaml
augmentation:
  random_resized_crop:
    scale: [0.7, 1.0]   # 扩大裁剪范围
    p: 0.7               # 提高概率
  channel_shuffle:
    enabled: true        # 启用通道混洗
    p: 0.2
```

### 3. 使用更大模型

- 从 `convnext_base` 升级到 `convnext_large` 或 `swin_large`
- 更大模型精度更高，但训练和推理速度更慢，显存占用更大
- 使用更大模型时可能需要减小 `batch_size`

```bash
python train.py --model convnext_large --batch_size 16
python train.py --model swin_large --batch_size 8
```

### 4. 调整损失权重

- 默认 `mae_weight=0.7`、`mse_weight=0.3`，侧重 MAE 优化
- 如果希望减少大误差样本，可增大 `mse_weight`（MSE 对大误差惩罚更重）
- 如果只关注平均精度，可增大 `mae_weight`

```bash
python train.py --model convnext_base --mae_weight 0.5 --mse_weight 0.5
```

### 5. 其他调优方向

- **增加训练轮数**：将 `--epochs` 增大至 300~500，配合早停机制自动停止
- **冻结骨干网络微调**：设置 `freeze_backbone: true`，先用小学习率微调头部，再解冻全模型训练
- **调整图像尺寸**：增大至 512 可提升精度，但显存占用显著增加
- **调整 Dropout**：过拟合时增大至 0.3~0.5，欠拟合时减小至 0.1
- **梯度裁剪**：训练不稳定时可减小 `max_norm` 至 0.5
