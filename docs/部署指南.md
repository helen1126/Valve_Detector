# 部署指南

本文档介绍阀门角度检测系统的生产环境部署方法，包括 API 服务、前端 Demo、ONNX 模型导出以及性能优化。

## 生产环境部署步骤

### 1. 安装依赖

```bash
# 创建独立环境
conda create -n valve_detector python=3.10 -y
conda activate valve_detector

# 安装 PyTorch（CUDA 12.1）
conda install pytorch torchvision pytorch-cuda=12.1 -c pytorch -c nvidia

# 安装项目依赖
pip install -r requirements.txt
```

### 2. 准备模型权重

将训练好的模型权重文件放置到 `./weights/` 目录下：

```bash
mkdir -p weights
# 将训练产出的 .ckpt 文件复制到 weights 目录
cp /path/to/trained/model.ckpt ./weights/last.ckpt
```

系统默认加载路径为 `./weights/last.ckpt`，可通过环境变量修改：

```bash
# 通过环境变量指定模型路径和架构
export MODEL_PATH=./weights/best.ckpt
export MODEL_NAME=convnext_base
export IMAGE_SIZE=384

# 远距离优化配置（默认开启智能裁剪）
export SMART_CROP=true
export MULTI_SCALE=false
```

**环境变量说明**：

| 环境变量 | 默认值 | 说明 |
|----------|--------|------|
| `MODEL_PATH` | `./weights/last.ckpt` | 模型权重路径 |
| `MODEL_NAME` | `convnext_base` | 模型架构名称 |
| `IMAGE_SIZE` | `384` | 输入图像尺寸 |
| `SMART_CROP` | `true` | 是否启用智能裁剪（远距离拍摄优化） |
| `MULTI_SCALE` | `false` | 是否启用多尺度推理（精度最高，速度稍慢） |

### 3. 验证模型可用

```bash
# 使用预测脚本验证模型加载和推理
python predict.py --model_path ./weights/last.ckpt --input ./dataset/top_view/0001_3.4.jpg

# 验证智能裁剪（远距离拍摄优化）
python predict.py --model_path ./weights/last.ckpt --input ./test.jpg --smart_crop

# 验证视频抽帧预测
python predict.py --model_path ./weights/last.ckpt --input ./test.mp4 --fps 2
```

## HTTPS 部署

当前端以 HTTPS 部署时，浏览器会拦截对 HTTP 接口的请求（混合内容策略）。此时需要启用 API 的 HTTPS 模式。

### 1. 生成 SSL 证书

#### 自签名证书（开发/内网测试）

**Linux/macOS：**

```bash
# 创建证书目录
mkdir -p certs

# 生成自签名证书（有效期 365 天）
openssl req -x509 -newkey rsa:4096 -keyout certs/key.pem -out certs/cert.pem -days 365 -nodes -subj "/CN=localhost"
```

**Windows（OpenSSL 配置文件缺失时）：**

如果 OpenSSL 报错 `Can't open "C:\Program Files\Common Files\ssl\/openssl.cnf"`，使用 Python 生成证书：

```powershell
# 创建证书目录
mkdir -p certs

# 使用 Python 生成证书
python -c "
from cryptography import x509
from cryptography.x509.oid import NameOID
from cryptography.hazmat.primitives import hashes, serialization
from cryptography.hazmat.primitives.asymmetric import rsa
from datetime import datetime, timedelta
import os

key = rsa.generate_private_key(public_exponent=65537, key_size=4096)
subject = issuer = x509.Name([x509.NameAttribute(NameOID.COMMON_NAME, 'localhost')])
cert = x509.CertificateBuilder().subject_name(subject).issuer_name(issuer).public_key(key.public_key()).serial_number(x509.random_serial_number()).not_valid_before(datetime.utcnow()).not_valid_after(datetime.utcnow() + timedelta(days=365)).sign(key, hashes.SHA256())

os.makedirs('certs', exist_ok=True)
with open('certs/key.pem', 'wb') as f:
    f.write(key.private_bytes(encoding=serialization.Encoding.PEM, format=serialization.PrivateFormat.TraditionalOpenSSL, encryption_algorithm=serialization.NoEncryption()))
with open('certs/cert.pem', 'wb') as f:
    f.write(cert.public_bytes(serialization.Encoding.PEM))
print('证书已生成')
"
```

如果没有 `cryptography` 库，先安装：
```powershell
pip install cryptography
```

#### Let's Encrypt 证书（生产环境，公网域名）

```bash
# 安装 certbot
pip install certbot

# 申请证书（需要 80 端口可用）
certbot certonly --standalone -d api.example.com

# 证书路径通常为：
# /etc/letsencrypt/live/api.example.com/fullchain.pem
# /etc/letsencrypt/live/api.example.com/privkey.pem
```

### 2. 配置环境变量

```bash
export SSL_KEYFILE=./certs/key.pem
export SSL_CERTFILE=./certs/cert.pem
export HTTPS_PORT=8443    # HTTPS 服务端口（默认 8443）
export HTTP_PORT=8000     # HTTP 重定向服务端口（默认 8000）
```

**HTTPS 环境变量说明**：

| 环境变量 | 默认值 | 说明 |
|----------|--------|------|
| `SSL_KEYFILE` | 无 | SSL 私钥文件路径（`.pem` 格式），设置后启用 HTTPS |
| `SSL_CERTFILE` | 无 | SSL 证书文件路径（`.pem` 格式），设置后启用 HTTPS |
| `HTTPS_PORT` | `8443` | HTTPS 服务监听端口 |
| `HTTP_PORT` | `8000` | HTTP 服务监听端口（与 HTTPS 并行提供完整 API） |

> **注意**：`SSL_KEYFILE` 和 `SSL_CERTFILE` 必须同时设置才会启用双服务模式。未设置时 API 以纯 HTTP 模式运行（仅 HTTP 端口，向后兼容）。

### 3. 启动 HTTPS 服务

```bash
# 方式1：直接运行（推荐，同时启动 HTTPS + HTTP 两个 API 服务）
python -m api.main

# 方式2：使用 uvicorn 命令行（仅 HTTPS 单端口）
uvicorn api.main:app --host 0.0.0.0 --port 8443 \
    --ssl-keyfile ./certs/key.pem \
    --ssl-certfile ./certs/cert.pem
```

启动后访问：
- HTTPS API：`https://localhost:8443/docs`
- HTTP API：`http://localhost:8000/docs`（与 HTTPS 提供相同接口，不重定向）

> **Cloudflare 隧道用户**：将隧道指向 HTTP 端口（`http://localhost:8000`）即可，Cloudflare 边缘会自动提供 HTTPS 给浏览器，无需配置后端 SSL 证书，也不会产生重定向循环。

### 4. 测试 HTTPS 服务

**Linux/macOS：**

```bash
# 跳过 SSL 验证测试
curl -k https://localhost:8443/health
```

**Windows PowerShell：**

PowerShell 的 `curl` 是 `Invoke-WebRequest` 的别名，不支持 `-k` 参数。使用以下方式：

```powershell
# 方法1：使用 PowerShell 原生命令（跳过 SSL 验证）
[System.Net.ServicePointManager]::ServerCertificateValidationCallback = {$true}
Invoke-WebRequest -Uri https://localhost:8443/health -UseBasicParsing

# 方法2：使用 curl.exe（Windows 内置）
curl.exe -k https://localhost:8443/health

# 方法3：如果 curl.exe 不在 PATH 中
& "C:\Windows\System32\curl.exe" -k https://localhost:8443/health
```

### 5. 前端调用 HTTPS API

当前端部署在 HTTPS 上时，必须使用 HTTPS 地址调用 API，否则浏览器会拦截混合内容：

```javascript
// 正确：HTTPS 前端调用 HTTPS API
const API_URL = "https://api.example.com:8443";

// 错误：HTTPS 前端调用 HTTP API（浏览器会拦截）
// const API_URL = "http://api.example.com:8000";
```

### 前端配置说明

在 Streamlit 界面侧边栏中配置：

1. **模型权重路径**：填写 `.ckpt`/`.pth`/`.onnx` 文件路径
2. **模型架构**：选择与权重匹配的模型架构
3. **图像尺寸**：选择输入图像尺寸（384/224/512）
4. **图像优化**：是否启用颜色增强、边缘检测等优化
5. **远距离优化**：
   - **智能裁剪**：远距离拍摄时自动定位并放大阀门区域（默认开启）
   - **多尺度推理**：结合原图和裁剪图预测，精度最高（默认关闭）
6. 点击 **加载模型** 按钮初始化

### 前端功能说明

前端提供三个功能 Tab：

- **📷 单张预测**：上传单张图片，显示预测角度和标注结果
- **📁 批量预测**：上传多张图片，显示结果表格和统计信息，支持 CSV 下载
- **🎬 视频预测**：上传视频文件，选择抽帧方式（按 fps 或帧间隔），显示角度变化折线图、结果表格，支持 CSV 和标注视频下载

## ONNX 模型导出

### 使用模型模块导出

```python
import torch
from models import build_model, export_onnx

# 加载模型
model = build_model(model_name="convnext_base", pretrained=False)
checkpoint = torch.load("./weights/last.ckpt", map_location="cpu", weights_only=False)

# 处理权重格式
if "state_dict" in checkpoint:
    state_dict = checkpoint["state_dict"]
    new_state_dict = {}
    for key, value in state_dict.items():
        if key.startswith("model."):
            new_state_dict[key[6:]] = value
        else:
            new_state_dict[key] = value
    model.load_state_dict(new_state_dict)
else:
    model.load_state_dict(checkpoint)

model.eval()

# 导出 ONNX
export_onnx(
    model=model,
    save_path="./weights/model.onnx",
    input_size=(1, 3, 384, 384),
    opset_version=17,
)
```

### ONNX 导出配置

在 `config/model_config.yaml` 中配置：

```yaml
onnx:
  enabled: true
  opset_version: 17
  dynamic_axes:
    input:
      0: batch_size
    output:
      0: batch_size
```

### 使用 ONNX 模型推理

```bash
python predict.py --model_path ./weights/model.onnx --input ./test.jpg --onnx
```

## 性能优化建议

### 1. 使用 ONNX Runtime 加速推理

ONNX Runtime 相比原生 PyTorch 推理通常有 2~3 倍加速：

```bash
# 导出 ONNX 模型后使用
python predict.py --model_path ./weights/model.onnx --input ./test.jpg --onnx
```

ONNX Runtime 自动选择可用的执行提供者（CUDA > CPU）。

### 2. 使用混合精度推理

API 服务默认使用 FP32 推理，可通过修改代码启用 FP16 推理以减少显存占用和提升速度。

### 3. 调整批大小

- 训练时：根据 GPU 显存调整 `batch_size`，ConvNeXt-Base + 384 尺寸在 24GB 显存上建议 `batch_size=32`
- 推理时：批量预测可适当增大批大小以提升吞吐量

### 4. 图像尺寸权衡

| 尺寸 | 精度 | 速度 | 显存 | 适用场景 |
|------|------|------|------|----------|
| 224 | 较低 | 最快 | 最小 | 快速验证 |
| 384 | 较高 | 中等 | 中等 | 推荐默认 |
| 512 | 最高 | 最慢 | 最大 | 精度优先 |

### 5. API 并发优化

- 使用 `--workers N` 启动多个 Uvicorn worker 进程
- Worker 数量建议设为 `CPU 核心数` 或 `2 × GPU 数量 + 1`
- 每个 Worker 会加载一份模型到显存，注意显存总量

### 6. 启用图像优化

对于光照条件不佳的场景，启用图像优化可提升预测精度：

```bash
python predict.py --model_path ./weights/last.ckpt --input ./test.jpg --optimize
```

图像优化流水线：光照校正（CLAHE + Gamma）→ 颜色增强 → 边缘增强

### 7. 远距离拍摄优化

当拍摄距离过远导致阀门在画面中占比过小时，启用智能裁剪或多尺度推理：

```bash
# 智能裁剪：自动检测阀门区域并裁剪放大（推荐，速度快）
python predict.py --model_path ./weights/last.ckpt --input ./test.jpg --smart_crop

# 多尺度推理：原图+裁剪图加权融合（精度最高，速度稍慢）
python predict.py --model_path ./weights/last.ckpt --input ./test.jpg --multi_scale
```

**阀门区域检测策略**（三级）：
1. 颜色特征检测（最可靠）：利用绿/红色掩码定位
2. 显著性检测（兜底）：频谱残差显著性
3. 边缘密度检测（最后手段）：网格化边缘密度分析

**自适应加权**：多尺度推理根据阀门占比自动调整权重，占比越小裁剪图权重越高。

### 8. 视频抽帧推理

支持视频文件抽帧预测，可选择按 fps 或帧间隔抽帧：

```bash
# 按每秒 2 帧抽帧
python predict.py --model_path ./weights/last.ckpt --input ./test.mp4 --fps 2

# 每 30 帧抽 1 帧，并输出标注视频
python predict.py --model_path ./weights/last.ckpt --input ./test.mp4 --frame_interval 30 --save_video

# 抽帧并保存标注帧为图片
python predict.py --model_path ./weights/last.ckpt --input ./test.mp4 --fps 1 --save_frames
```

输出文件：
- `video_predictions.csv`：帧索引、时间戳、预测角度、处理时间
- `predicted_video.mp4`：带角度标注的视频（`--save_video` 时）
- `frame_XXXXXX_XX.X.jpg`：标注帧图片（`--save_frames` 时）

## 常见部署问题解决

### 端口被占用

**现象**：启动 API 或前端时提示端口已被占用。

**解决方法**：

```bash
# Windows：查找并终止占用端口的进程
netstat -ano | findstr :8000
taskkill /PID <进程ID> /F

# 或更换端口
uvicorn api.main:app --host 0.0.0.0 --port 8001
streamlit run frontend/app.py --server.port 8502
```

### 模型加载失败

**现象**：API 启动时提示模型加载失败。

**排查步骤**：

1. 检查模型文件路径是否正确
2. 检查模型架构名称是否与权重匹配（`MODEL_NAME` 环境变量）
3. 检查文件权限
4. 查看详细错误日志

```bash
# 检查文件是否存在
ls -la ./weights/last.ckpt

# 指定正确的模型架构
export MODEL_NAME=convnext_base
export MODEL_PATH=./weights/last.ckpt
```

### CUDA 不可用

**现象**：模型在 CPU 上运行，推理速度慢。

**排查步骤**：

1. 检查 NVIDIA 驱动是否安装：`nvidia-smi`
2. 检查 CUDA 版本是否匹配：`nvcc --version`
3. 检查 PyTorch CUDA 是否可用：

```python
import torch
print(torch.cuda.is_available())
print(torch.version.cuda)
```

4. 重新安装匹配版本的 PyTorch

### 显存不足

**现象**：启动时或推理时 OOM（Out of Memory）。

**解决方法**：

1. 减小 `IMAGE_SIZE`（如 384 → 224）
2. 减小 API worker 数量
3. 使用 ONNX 模型推理（显存占用更小）
4. 使用更小的模型架构（如 `resnet50` 替代 `convnext_base`）

### API 响应慢

**排查方向**：

1. 确认是否使用 GPU 推理（检查 `/info` 接口返回的 `device` 字段）
2. 检查图像优化是否开启（会增加处理时间）
3. 考虑使用 ONNX Runtime 加速
4. 检查网络带宽和图片大小
